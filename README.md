# DBchangePakets

Приложение представляет собой REST-микросервис на Go для управления пользователями (регистрация и получение информации). Оно спроектировано как демонстрационный/шаблонный проект с поддержкой двух различных систем управления базами данных (СУБД) — **PostgreSQL** и **MongoDB** — переключение между которыми осуществляется через конфигурацию без изменения бизнес-логики.

## Какую задачу решает этот код?

Основная задача проекта — продемонстрировать гибкую архитектуру Go-приложения, которая позволяет абстрагироваться от конкретной СУБД. Код решает следующие инфраструктурные и архитектурные задачи:

1. **Двухплатформенное хранилище (Multi-Database Support):**
   Реализована возможность прозрачного переключения между реляционной БД (**PostgreSQL**) и документоориентированной NoSQL БД (**MongoDB**) с помощью конфигурационной переменной `DB_TYPE` или флага `-db`.
   
2. **Чистая архитектура (Decoupling & Interface Segregation):**
   * **Слой домена (`internal/domain`):** содержит независимые структуры данных (модель `User`).
   * **Бизнес-логика (`internal/user`):** полностью изолирована. Интерфейс `UserRepository` определен в пакете `user` (на стороне потребителя), что предотвращает завязку на конкретную технологию хранения.
   * **Слой инфраструктуры (`internal/repository`):** содержит конкретные реализации интерфейса для PostgreSQL (с сырыми SQL-запросами без ORM) и MongoDB (с использованием официального `mongo-driver`).
   
3. **Автоматическое управление схемой данных:**
   * **PostgreSQL:** при старте приложения автоматически применяются миграции из папки `internal/repository/postgres/migrations` с помощью механизма `go:embed` и библиотеки `golang-migrate`.
   * **MongoDB:** при старте автоматически проверяется и создается уникальный индекс на поле `email` для гарантии уникальности записей.

4. **Двенадцатифакторное конфигурирование (12-Factor App Config):**
   Параметры приложения (порты, строки подключения, лимиты пулов) считываются из переменных окружения с возможностью переопределения через флаги командной строки.

5. **Graceful Shutdown (Безопасное завершение работы):**
   Приложение перехватывает сигналы завершения работы ОС (`SIGINT`, `SIGTERM`), прекращает принимать новые HTTP-запросы, дожидается завершения обработки текущих запросов и корректно закрывает соединения с базами данных (пулы соединений) в порядке, обратном их созданию (LIFO).

## Стек технологий

* **Язык программирования:** Go (Golang)
* **HTTP Роутинг:** Стандартный `net/http` (с поддержкой маршрутизации из Go 1.22+)
* **Логирование:** Структурированный логгер `log/slog` (вывод в JSON-формате)
* **Базы данных:**
  * **PostgreSQL** (драйвер `github.com/lib/pq`, миграции `golang-migrate`)
  * **MongoDB** (официальный драйвер `go.mongodb.org/mongo-driver`)
* **Тестирование:** Встроенный пакет `testing` (тестирование бизнес-логики через in-memory фейк-репозиторий без внешних зависимостей)

## API Эндпоинты

Приложение предоставляет следующие HTTP REST эндпоинты:

* `POST /users` — Регистрация нового пользователя.
  * **Тело запроса:** `{"username": "ivan", "email": "ivan@example.com"}`
  * **Ответ (201 Created):** Данные созданного пользователя (включая сгенерированный ID и дату создания).
  * **Ошибки:** `400 Bad Request` (невалидные данные, пустой username), `409 Conflict` (email уже занят).
* `GET /users/{id}` — Получение пользователя по его уникальному идентификатору.
  * **Ответ (200 OK):** Данные пользователя.
  * **Ошибки:** `404 Not Found` (пользователь не найден).
* `GET /health` — Проверка работоспособности сервиса.
  * **Ответ (200 OK):** `{"status":"healthy"}`

## Конфигурация и запуск

Приложение настраивается с помощью переменных окружения или флагов командной строки:

| Переменная окружения | Флаг командной строки | Значение по умолчанию | Описание |
|---|---|---|---|
| `PORT` | `-port` | `8080` | Порт HTTP-сервера |
| `DB_TYPE` | `-db` | `mongodb` | Выбор базы данных (`postgres` или `mongodb`) |
| `PG_URI` | `-pg-uri` | `postgres://postgres:postgres@localhost:5432/demo?sslmode=disable` | Строка подключения к PostgreSQL |
| `PG_MAX_OPEN` | `-pg-max-open` | `25` | Максимальное количество открытых соединений в пуле PG |
| `PG_MAX_IDLE` | `-pg-max-idle` | `25` | Максимальное количество простаивающих соединений в пуле PG |
| `PG_LIFETIME_MIN` | `-pg-lifetime-min` | `5` | Время жизни соединения в пуле (в минутах) |
| `MONGO_URI` | `-mongo-uri` | `mongodb://localhost:27017` | Строка подключения к MongoDB |
| `MONGO_DBNAME` | `-mongo-dbname` | `demo_db` | Имя базы данных MongoDB |

### Запуск с MongoDB (по умолчанию)
```bash
go run cmd/server/main.go
```

### Запуск с PostgreSQL
Сначала убедитесь, что база данных запущена, затем укажите нужные переменные окружения или флаги:
```bash
export DB_TYPE=postgres
export PG_URI="postgres://user:password@localhost:5432/my_db?sslmode=disable"
go run cmd/server/main.go
```
или с использованием флагов:
```bash
go run cmd/server/main.go -db postgres -pg-uri "postgres://user:password@localhost:5432/my_db?sslmode=disable"
```
